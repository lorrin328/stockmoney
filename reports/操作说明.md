## 一、文档结构与用途说明

本系统现有两份操作类文档,分工如下：

| 文档 | 定位 | 适用场景 |
|------|------|---------|
| `reports/OPERATION_MANUAL.md` | **技术操作手册** | 命令行工具的详细用法、参数、回测、模块组合 |
| `reports/操作说明.md`(本文档) | **用户工作流说明** | 用户通过自然语言触发的三大闭环功能 |

两者关系：本文档面向**用户与 AI 的对话工作流**,OPERATION_MANUAL 面向**脚本与命令行**。三大功能的底层均依赖 OPERATION_MANUAL 中描述的脚本模块。

## 二、系统总览

### 2.1 系统定位

StockMoney 是一个**持续学习、迭代的个人 ETF 投资决策系统**,具有三个核心特性：

1. **可进化** — 模型可由用户命令触发重新研究、迭代、校验
2. **闭环建议** — 用户上传持仓截图后,系统结合实际持仓给出操作建议
3. **周期复盘** — 周末/月末输出持仓分析报告,沉淀经验

### 2.2 三大用户触发功能一览

| 功能 | 触发方式 | 输入 | 核心输出 | 输出位置 |
|------|---------|------|---------|---------|
| **F1 模型自学习与校验** | 用户命令 | 自然语言指令 | 迭代后的模型代码/研究报告 | `papers/`、`scripts/`、`reports/research_*.md` |
| **F2 持仓记录与建议** | 上传截图 + 询问 | 持仓截图 + 用户问题 | 持仓 JSON + 操作建议报告 | `data/positions/`、`reports/advice_*.md` |
| **F3 持仓分析报告** | 用户要求总结 | 时间范围(周/月) | 持仓复盘报告 | `reports/portfolio_review/` |

### 2.3 与已有 5 层架构的关系

```
已有底座:
  Layer 1 康波周期定位 (kondratiev_model.py)
  Layer 2 周期共振 + 市场指标 (cycle_phase_evaluator.py + market_indicators.py)
  Layer 3 资产配置决策 (asset_allocator.py)
  Layer 4 策略决策引擎 (strategy_engine.py)
  Layer 5 4%定投法执行 (four_percent_model.py)
  + 政策分析 (policy_analyzer.py)
  + 每日监控 (investment_monitor.py)
  + ETF 多维度甄选 (etf_selector.py)
  + 研究驱动器 (research_driver.py)

新增三大功能(本文档):
  F1 → 调用并迭代 Layer 1-5 的模型 + 写入新研究
  F2 → 读取 Layer 4-5 的当前决策,叠加用户实际持仓数据
  F3 → 综合 Layer 1-5 输出 + 持仓收益归因
```

## 三、功能 1：模型自学习与校验

### 3.1 触发方式

**用户用自然语言下达命令**,典型指令包括(但不限于)：

| 用户说法 | 含义 |
|---------|------|
| "更新模型"、"迭代模型" | 全面更新所有底层模型(康波/周期/政策/资产配置/4%定投) |
| "学习"、"做研究"、"研究一下 XXX" | 针对特定主题做深度研究并沉淀 |
| "校验模型"、"回测"、"复盘模型" | 用历史数据校验模型有效性 |
| "更新 [模块名]" | 仅更新指定模块,如"更新政策"、"更新康波" |
| "获取最新 [主题]" | 收集某一主题的最新信息(如政策、宏观、行业) |

### 3.2 Claude 执行流程

```
步骤 1: 识别意图
  └── 解析用户指令,确定:
      ├── 是全量迭代还是定向学习?
      ├── 涉及哪些模块?(康波/周期/政策/资产配置/4%定投/ETF评分)
      └── 是否需要回测验证?

步骤 2: 信息收集
  └── 工具:
      ├── WebSearch / WebFetch (最新政策、宏观数据、研究报告)
      ├── MCP 服务器 (alpha-vantage / investor-agent / helium 等)
      ├── 已有 papers/、reports/ (历史研究沉淀)
      └── 已有 scripts/ (现有模型代码)

步骤 3: 模型迭代或校验
  ├── 迭代:更新模型参数、阶段判断、配置矩阵 → 写入对应 .py 文件
  ├── 校验:用历史数据回测 → 输出 backtest_*.md 报告
  └── 研究:沉淀深度研究报告 → 写入 papers/ 或 reports/

步骤 4: 沉淀输出
  ├── 模型代码更新: scripts/*.py(直接修改)
  ├── 研究报告: papers/{主题}_{YYYYMMDD}.md
  ├── 迭代日志: reports/research_{YYYYMMDD}.md (记录"改了什么、为什么改、依据是什么")
  └── 配置更新: data/portfolio_config.json (若涉及 ETF 评分变动)
```

### 3.3 输出位置规范

| 输出类型 | 路径 | 命名 |
|---------|------|------|
| 深度研究 | `papers/` | `{主题}_{YYYYMMDD}.md` |
| 模型迭代日志 | `reports/` | `research_{YYYYMMDD}.md` |
| 政策分析 | `reports/` | `policy_analysis_{YYYYMMDD}.md` |
| 回测报告 | `reports/` | `backtest_{code}_{YYYYMMDD}.md` |
| 模型代码 | `scripts/` | 直接修改原 `.py` 文件 |
| ETF 评估 | `reports/` | `etf_evaluation_{YYYYMMDD}.md` |

### 3.4 使用示例

**示例 A：全量迭代**

> 用户："请更新所有模型,基于最新政策面和宏观数据"

Claude 执行：
1. WebSearch 最新政策(十五五细则、央行政策、美联储动向)
2. 更新 `policy_analyzer.py` 中的政策判断
3. 更新 `kondratiev_model.py` 当前阶段判断
4. 更新 `asset_allocator.py` 阶段配置矩阵(若需)
5. 写入 `reports/research_20260502.md`,记录所有变更
6. 提示用户:模型已更新,变更摘要见研究日志

**示例 B：定向研究**

> 用户："研究一下黄金 2026 下半年的走势"

Claude 执行：
1. 收集 2026 年至今黄金价格、央行购金、地缘事件、美元走势
2. 写入 `papers/黄金2026下半年研究_20260502.md`
3. 若研究结论影响 ETF 评分,顺带更新 `data/portfolio_config.json`

**示例 C：模型校验**

> 用户："对沪深 300 跑一下 4%定投回测"

Claude 执行：
1. 运行 `python scripts/four_percent_model.py --backtest 510300 --start 2022-01-01 --capital 100000`
2. 输出已自动写入 `reports/backtest_510300_{YYYYMMDD}.md`
3. 给用户摘要:总收益、最大回撤、触发次数、与买入持有对比

## 四、功能 2：持仓截图记录与建议

### 4.1 触发方式

**用户上传持仓截图后,通过询问触发**,典型指令包括：

| 用户说法 | 含义 |
|---------|------|
| "我现在该买什么?" | 基于当前持仓 + 现价给出加仓/减仓建议 |
| "给我建议"、"看看我的持仓" | 综合分析持仓健康度并给出操作 |
| "这只 [代码] 该不该卖?" | 针对单一标的给出止盈/止损建议 |
| "帮我对照一下目标配置" | 检查实际持仓与目标权重的偏离 |
| (仅上传截图,不发问) | 默认仅记录持仓,不给建议(等用户后续询问) |

### 4.2 Claude 执行流程

```
步骤 1: 截图读取(视觉识别)
  └── Claude 直接读取图片中的持仓信息,提取:
      ├── ETF/股票代码(必需)
      ├── ETF/股票名称(必需)
      ├── 持仓份额(必需)
      ├── 持仓成本(必需)
      ├── 当前市值/现价(必需)
      ├── 浮动盈亏(可选,可推算)
      └── 截图日期(可选,默认当日)

步骤 2: 数据规整与持久化
  ├── 标准化为持仓 JSON 结构(见 4.3)
  ├── 写入 data/positions/positions_{YYYYMMDD}.json
  ├── 同步更新 data/portfolio_config.json 的 positions 字段
  └── 记录到 data/positions/positions_history.csv (历史快照,便于复盘)

步骤 3: 偏离分析
  ├── 对照 data/portfolio_config.json 的 target_pct
  ├── 计算每个标的的:实际权重、目标权重、偏离百分点
  └── 标记: ⚠️ >+5%(超配)、⚠️ <-5%(欠配)、✅ ±5% 内(合理)

步骤 4: 建议生成
  ├── 调用 strategy_engine 获取当前阶段决策
  ├── 调用 four_percent_model 检查 4%定投触发情况
  ├── 调用 investment_monitor 拉取实时分位与 E/P
  └── 综合输出:
      ├── 紧急操作(止盈/止损/触发买入)
      ├── 关注操作(估值偏离、政策事件影响)
      └── 持有不动(健康仓位)

步骤 5: 报告输出
  └── reports/advice_{YYYYMMDD}.md
      包含:
        ├── 持仓概览(代码/名称/份额/成本/现值/浮盈/权重)
        ├── 偏离分析(超配/欠配标的清单)
        ├── 操作建议(分紧急/关注/持有三档)
        ├── 4%定投触发情况
        └── 风险提示
```

### 4.4 输出位置规范

| 文件 | 路径 | 用途 |
|-----|------|------|
| 持仓快照 | `data/positions/positions_{YYYYMMDD}.json` | 单日持仓 JSON |

### 4.5 使用示例

**示例 A：上传截图询问建议**

> 用户:[上传持仓截图] "我现在该买什么?"

Claude 执行：
1. 读取截图,提取持仓 → 写入 `data/positions/positions_20260502.json`
2. 同步到 `data/portfolio_config.json`
3. 调用 strategy_engine + four_percent_model
4. 输出 `reports/advice_20260502.md`,内容如:
   - 黄金 ETF 实际 8% / 目标 12.6%,**欠配 4.6pp**,建议加仓
   - 芯片 ETF 触发 4%定投买入信号,建议买入 1 份
   - 创新药 ETF 浮盈 25%,接近止盈线,建议关注

**示例 B：仅上传截图(不询问)**

> 用户:[上传持仓截图]

Claude 执行：
1. 读取截图 → 写入 `data/positions/positions_20260502.json`
2. 回复:"已记录 X 只标的持仓,总市值 XX 万。如需操作建议请告知。"
3. **不主动给建议**(等用户后续询问)

**示例 C：单标的咨询**

> 用户:[已有持仓数据] "创新药 ETF 该不该卖?"

Claude 执行：
1. 读取最新持仓 JSON 中的 159992 数据
2. 调用 market_indicators 获取当前估值分位
3. 检查 4%定投的卖出条件(E/P < 6.4%)
4. 给出明确建议:卖出/持有/部分止盈

## 五、功能 3：周期性持仓分析报告

### 5.1 触发方式

**用户在周末或月末要求总结时触发**,典型指令包括：

| 用户说法 | 含义 |
|---------|------|
| "周报"、"做个周报"、"本周总结" | 生成本周持仓分析报告 |
| "月报"、"做个月报"、"本月总结" | 生成本月持仓分析报告 |
| "总结一下"、"复盘一下" | 默认按本周(若周末)或本月(若月末)生成 |
| "做一份 [日期] 到 [日期] 的报告" | 自定义时间范围 |

### 5.2 Claude 执行流程

```
步骤 1: 确定时间范围
  ├── 周报: 上周一至本周日
  ├── 月报: 本月 1 日至月末
  └── 自定义: 用户指定区间

步骤 2: 数据汇集
  ├── 读取期初/期末持仓快照(data/positions/)
  ├── 读取期间所有信号报告(data/signals/)
  ├── 读取期间所有建议报告(reports/advice_*.md)
  └── 拉取期间各标的价格走势(data/history/ 或实时拉取)

步骤 3: 收益归因
  ├── 总账: 期初市值 → 期末市值,绝对收益与百分比
  ├── 单标的归因: 每只 ETF 的收益贡献度
  ├── 行业归因: 按 ETF 类别聚合(宽基/进攻/防御/卫星)
  └── 操作归因: 已执行操作的盈亏分析

步骤 4: 偏离与策略对照
  ├── 实际权重 vs 目标权重
  ├── 实际操作 vs 模型建议(几次触发,几次执行)
  └── 与基准的对比(沪深300/中证A500等)

步骤 5: 下阶段策略
  ├── 调用 strategy_engine 获取下周/下月策略
  ├── 列出待办操作清单(再平衡、定投执行)
  └── 风险提示(政策/估值/流动性)

步骤 6: 报告输出
  └── reports/portfolio_review/{周报|月报}_{YYYYMMDD}.md
```

### 5.3 报告标准结构

```markdown

### 2.4 操作归因(本期所有买卖记录)

### 3.2 超配/欠配标的清单

## 四、模型与策略复盘
### 4.1 本期 4%定投触发情况

## 五、宏观与政策回顾
### 5.1 本期重要政策/经济事件
### 5.2 对持仓的影响评估

## 六、下阶段策略
```

## 七、AI 助手工作约束

### 7.1 触发原则

1. **三大功能均由用户主动触发**,Claude 不自动定时执行
2. **功能 2 必须先有截图**才能给建议(无截图则要求用户上传)
3. **功能 3 默认按当前时点判断**(周末→周报,月末→月报),除非用户指定区间

### 7.2 输出原则

参照 `souls.md` 第一章 8 项原则:
1. **结论先行** — 报告开头即给出核心结论
2. **结构化输出** — 采用"总—分""问题—原因—对策"框架
3. **不空泛** — 避免"适当、持续优化、逐步提升"等模糊表达
